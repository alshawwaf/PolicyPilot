Reuse first

No-op

If a rule already permits the traffic, it's a no-op — just attach it to the ticket. No change to live policy.

Widen only on an exact match

Widen

Extend an existing rule only when it already matches the request in the other two columns — so it never grants access nobody asked for.

Otherwise create least-privilege

Create

A fresh, narrow rule for exactly the request — correctly placed so the access actually takes effect.

Worked example

Match by kind

Resolve

Each endpoint is read by its kind — an IP/CIDR/Any by address, a typed object by identity — so the two never get confused.

Worked example

Note, don't stop

Note

A rule it can't fully resolve is flagged “review later” and the walk continues — and the new allow is placed below it, so first-match keeps that unresolved rule in effect.

Worked example

Dynamic layers are out of scope

Skip

Rules applying a dynamic inline layer are skipped from matching — they're managed out-of-band on the gateway.

Worked example

Only a broken request stops it

Fix & retry

If the request itself can't be turned into a change — an empty or unresolvable cell — it's flagged for a quick fix. The only thing that ever stops the engine.
