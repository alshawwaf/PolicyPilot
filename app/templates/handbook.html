Overview

PolicyPilot is the front door to Check Point access automation. Instead of hand-editing rulebases, you describe the access you want — in the portal, from a ticket, or through an LLM agent — and the engine computes the minimal, first-match-safe change, previews it, and applies it on approval with one-click rollback. The UI is a macOS-style desktop: every tool is an icon on the dock and desktop, and each opens in a draggable, resizable window.

The decision engine

Reuse, widen, or create — computed against the live rulebase, placed above the right deny under first-match, reusing existing objects instead of minting duplicates.

Two rails

Apply to the Security Management Server over the Management web_api, or push a dynamic layer straight to a gateway via the Gaia API — each with its own publish gate.

Four ways to drive it

An MCP agent (29 tools), the REST API, a ticket webhook, or the portal UI — one decision engine behind all of them.

Under the hood it's a FastAPI + SQLite app with a Jinja / vanilla-JS desktop shell, validated against a live Check Point R82.10 Management Server. Each signed-in user gets their own desktop layout; admins curate the shared default.

Using the desktop

The portal behaves like a real desktop OS — the interactions you already know from macOS work here.

Windows

Open a tool from the dock or a desktop icon; drag it by the title bar, resize from any edge, and minimize / zoom / close with the traffic-light controls. Stack as many as you like.

Dock & Launchpad

The dock holds your pinned apps; the Launchpad lists every tool with Dock / Desktop toggles. Right-click any icon for open, pin, and add-to-desktop.

Your layout, saved

Drag desktop icons to arrange them; your dock and desktop persist per user. Switch dark / light theme from the menubar — the whole portal follows.

Access automation

The core. State a request as source → destination : service and the engine decides what the minimal change is — then previews it before anything is written:

No-op

The access already exists — a rule permits it today. Nothing to change; the engine says so and shows which rule.

Widen

An existing rule (or a group it references) can be extended to cover the request — the least-change grant, instead of a new rule.

Create

A new rule is needed. It's placed first-match-safe — above the right deny, below the right stealth / cleanup, in the right section.

Every rule column is writable — action (Accept / Drop / Reject / Ask / Inform / Apply Layer) with Action Settings / UserCheck, content, time, install-on, VPN, and a bandwidth limit. Discovery resolves a plain phrase — "the finance role", "work hours", "SQL Queries" — to the real Check Point object, or offers ranked "did you mean" candidates. Apply publishes on approval, and every published change is reversible in one click.

The Access automation app runs the full flow; Field support lists exactly which object types each column accepts. For the reasoning in depth, see the access-automation white paper in the repository.

Interfaces & API

The same decision engine is reachable four ways — all authenticated, all honouring the same publish gates (reads are always allowed; a live write needs the gate).

MCP — 29 tools

Connect n8n, Claude Desktop, Cursor, or any MCP client over /mcp with an mcp-scope key. Onboarding and the tool catalog live in the Agents & MCP app.

REST API

The same engine at /dbapi/v1 (api-scope key). Browse it in the PolicyPilot API app and try calls live in the API explorer.

Ticket webhook

A ServiceNow / Jira / any webhook becomes a Check Point rule, authenticated with X-PolicyPilot-Token, with optional write-back of the outcome.

Deploying

PolicyPilot is a single container: it listens on port 8000, stores state in SQLite under /data, and sits behind a TLS-terminating reverse proxy. Pick the method that fits your environment:

Docker Compose

The bundled compose file runs the app plus Caddy for automatic HTTPS. cp .env.example .env, then docker compose up -d --build.

Plain Docker

docker build + docker run behind any reverse proxy (nginx, Caddy, Traefik) or cloud load balancer that forwards to port 8000.

A container PaaS

Dokploy, Coolify, Render, Fly.io — build from the Dockerfile, expose 8000, add a domain, and mount a volume at /data.

Whatever the method, persist PILOT_ENCRYPTION_KEY (it encrypts saved credentials) and set PILOT_TRUSTED_PROXY_HOPS to the number of proxies in front. The full guide — the complete PILOT_* reference and the single-worker note — is in DEPLOY.md.

Administration

Admins run the portal and curate what everyone sees.

Users & roles

Multi-user with granular capabilities (preview / apply / publish / export / manage-users). Self-signup lands pending until an admin approves. Manage from Users & Groups.

Settings & secrets

Every integration secret (MCP token, webhook, ServiceNow) is set from Settings, encrypted at rest — no redeploy. The publish gate for each rail lives here too.

Health & audit

The System app shows live health; /version carries the build SHA; every committed change raises a governance event, and Activity logs every call.
