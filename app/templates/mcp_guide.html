MCP for agents

Connect an LLM agent (n8n, Claude Desktop, Cursor, VS Code) to PolicyPilot over the Model Context Protocol. The agent gets typed tools for two things: automate your Check Point management (SMS) access policy via the Management Web API, and author + push dynamic layers to a gateway via the Gaia API — each write behind its own admin gate. Same idea as Check Point's MCP servers, focused on the access-automation engine.

Safety

  

  
Every call needs Authorization: Bearer <key>, where the bearer is an
  active MCP-scope API key (generated above, or in Settings → API keys — individually
  revocable, SHA-256-hashed at rest). Create or revoke takes effect with no redeploy — no active key returns a
  503 (endpoint disabled), a wrong or missing key returns a 401. decide_access, the analyze
  tools, and the correlate and coverage tools are read-only. apply_access with publish=false
  dry-runs (validate, then discard); publish=true is refused unless an admin enables
  Settings → MCP / agent → "Let the MCP agent publish to live policy". The engine's own guarantees
  still hold — an unknown or ambiguous service returns a review plus suggestions, never a wrong call to the SMS.
