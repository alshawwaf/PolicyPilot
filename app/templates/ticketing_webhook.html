Ticketing webhook

Wire any ITSM — ServiceNow, Jira, Remedy, Freshservice, or a custom portal — straight to PolicyPilot: POST a ticket as JSON and it decides (and, with "apply": true, applies + publishes) end-to-end. One of three ways to drive PolicyPilot, alongside agents (MCP) and the REST API.

Point any system that can POST JSON at the endpoint below. Set PILOT_WEBHOOK_TOKEN (Settings → integrations) and send it as the X-PolicyPilot-Token header — the webhook is disabled until that token is set, so it never runs unauthenticated. Field names are flexible (common u_* / number / Jira key aliases are accepted). The result is returned synchronously; to push it back, add an optional "callback_url" (we POST the result there) or configure the built-in ServiceNow write-back (PILOT_SERVICENOW_*).

Example request
POST {{ request.base_url }}access-automation/webhook
X-PolicyPilot-Token: <your token>
Content-Type: application/json

{
  "ticket_id": "INC0012345",
  "server_id": {{ example_server_id }},
  "layer": "<layer name>",
  "source": "192.168.9.9",
  "destination": "172.16.5.10",
  "protocol": "tcp",
  "port": "443",
  "apply": false,
  "callback_url": "https://itsm.example/api/result"
}
